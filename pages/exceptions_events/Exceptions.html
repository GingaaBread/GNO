Exceptions

Exceptions signalise exceptional situations occuring in a program. When they occur, the program will be instantly terminated completely disrupting the normal program flow.

Example situations could be:

  • A variable has a null value, but the programmer attempts to access a field
  • A programmer calls a method with a negative method argument, but the method requires positive arguments
  • A programmer attempts to access the seventh item of an array with only four elements

Throwing Exceptions

The throw keyword is used to throw an exception. 

                
                    main class GNOTest {
                        throw Exception()
                    }
Output:
gno.err.Exception: at GNOTest.main(2)

gno.err.Exception means that the class Exception is used, which is located in the gno.err package.

After the colon : follows the description, which is empty in this example.

at GNOTest.main(2) shows that the exception occured in the method called main in the class called GNOTest at line 2.

You can add a string argument for the description:

                
                    main class GNOTest {
                        throw Exception("The server is not yet running.")
                    }
Output:
gno.err.Exception: "The server is not yet running." at GNOTest.main(2)

When using the default Exception class with a description, you can emit the Exception part: 

                
                    main class GNOTest {
                        throw "The server is not yet running."
                    }
Output:
gno.err.Exception: "The server is not yet running." at GNOTest.main(2)

It is recommended to use exceptions that fit semantically to the respective problem. You can use a multitude of predefined exceptions.

Predefined Exceptions
Package Name Description Example
gno.err.Exception Exception The default exception Any generic case
gno.err.OutOfBoundsException OutOfBoundsException A value is not within bounds of a certain range Trying to get the first element of an empty list
gno.err.IllegalArgumentException IllegalArgumentException One or more arguments of a called method are not allowed Passing a string with too many characters
gno.err.IllegalOperationException IllegalOperationException An operation is not allowed Dividing by zero
gno.err.CastException CastException A cast cannot be done Illegally casting a class to another class
gno.err.IllegalStateException IllegalStateException An operation is called at an inappropriate time or in a wrong state Accessing a closed stream
gno.err.NullAccessException NullAccessException A null value is accessed Not instantiating an object, but trying to call its method
gno.err.NotImplementedException NotImplementedException A functionality is not (yet) implemented Calling a not yet fully finished method
gno.err.StackOverflowException StackOverflowException An overflow has occured on the stack Using recursion, but forgetting the exit condition
gno.err.IllegalFormatException IllegalFormatException A type does not adhere to a certain format Passing a date in the DD/MM/YYYY format instead of the YYYY/MM/DD format 
                
                    main class GNOTest {
                        ExampleClass exampleClass
                        exampleClass.foo()
                    }
Output:
gno.err.NullAccessException: "Accessing a value with a null type." at GNOTest.main(3)

IllegalArgumentException

You can automatically check for illegal method arguments by adding a preset or custom boolean to a method parameter. If the method is called with an argument, which does not adhere to the restriction, an IllegalArgumentException will be thrown.

Presets:

  • positive / + (#>0)
  • negative / - (#<0)
  • notnull / # (#!=null)
  • notempty / | (|#|>0)
                
                    class GNOTest {
                        void Main() {
                            TransferMoney(5,-1.5d,null,"AB3937806");  
                        }

                        // Presets
                        void TransferMoney(int amount:positive, double taxeReduction:-, Account account:#, string id:|) {}
                    
                        // Custom Boolean Expression
                        void CustomExample(int:#>5&&#<2) {}
                    }
Output:
gno.err.IllegalArgumentException: "Argument 'account' must not be null" at GNOTest.main(2)
